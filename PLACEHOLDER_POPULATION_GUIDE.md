# üìù Understanding Slide Layouts & Content Positioning

## How Content Works with Custom Slide Layouts in PptxGenJS

---

## ‚ö†Ô∏è **CRITICAL UNDERSTANDING**

### **What "Layouts" Actually Mean in This System**

The 55 custom slide layouts in this repository:
- ‚úÖ **Provide visual design** (logos, backgrounds, styling)
- ‚úÖ **Are referenced by slides** (slide links to layout XML)
- ‚ùå **DO NOT have editable placeholders** in the traditional PowerPoint sense
- ‚ùå **Content is NOT inherited** from layout placeholder positions

### **How PptxGenJS Actually Works**

When you create a slide with `{ masterName: "Two Content" }`:

1. **The slide references the layout XML** (appears in slide relationships)
2. **The layout provides the visual template** (logos, colors, backgrounds)
3. **BUT: You manually add ALL content** to the slide as independent shapes
4. **Content is positioned using absolute coordinates**, not placeholders

### **What This Means**

```javascript
// ‚úÖ This creates a slide that REFERENCES the layout
const slide = pptx.addSlide({ masterName: "Two Content" });

// ‚ùå But placeholders DON'T work like PowerPoint
slide.addText("Hello", { placeholder: "title" }); // Does nothing!

// ‚úÖ You must position content manually
slide.addText("Hello", { 
  x: 0.62,  // Manual positioning
  y: 0.38, 
  w: 12.1, 
  h: 0.5 
});
```

**Result:** The slide gets the layout's visual design (logos, styling) + your manually positioned content.

---

## üéØ **What You Get from Layouts**

When you use `{ masterName: "LayoutName" }`:

### ‚úÖ **You DO Get:**
1. **Visual design** - Background colors, borders, decorative elements
2. **Pre-positioned logos** - 4 logo variants embedded in layout
3. **Consistent styling** - Colors, fonts defined in theme
4. **Professional appearance** - Layout matches comprehensive-test.pptx design

### ‚ùå **You DON'T Get:**
1. **Editable placeholders** - No "Click to add text" boxes
2. **Auto-positioned content** - Content doesn't flow into preset areas
3. **Inherited coordinates** - You must specify x, y, w, h for everything
4. **Placeholder text replacement** - Can't use `{ placeholder: "name" }`

The slide:
- **References** the layout (for visual design)
- **Contains** all content as independent shapes
- **Does NOT** use layout placeholders
- **Requires** manual positioning of every element

**This is fundamentally different from PowerPoint's placeholder system.**

## ÔøΩüìê **How to Position Content Correctly**

### **Step 1: Select Your Layout**

```javascript
import pptxgen from 'pptxgenjs';

const pptx = new pptxgen();
const slide = pptx.addSlide({ masterName: "Two Content" });
```

### **Step 2: Get Coordinates from `tools/step_templates.llm.json`**

The layout XML defines **visual elements** (logos, backgrounds), but your **content must be positioned manually**.

The file `tools/step_templates.llm.json` provides **suggested coordinates** that align with the layout's visual design.

**These are NOT placeholders - they're coordinate suggestions for where content looks good visually.**

**Example for "Two Content" layout:**
```json
{
  "name": "Two Content",
  "masterName": "Two Content",
  "code": [
    "const slide = pptx.addSlide({ masterName: \"Two Content\" });",
    "slide.addText(ctx.title, { x: 0.62, y: 0.38, w: 12.1, h: 0.5 })",
    "slide.addText(ctx.subHeadline, { x: 0.62, y: 0.98, w: 12.1, h: 0.31 })",
```

## üõ†Ô∏è **Helper Function (Recommended)**

Create a helper to load coordinates automatically:

```javascript
import fs from 'fs';
import pptxgen from 'pptxgenjs';

// Load templates once
const templates = JSON.parse(
  fs.readFileSync('tools/step_templates.llm.json', 'utf8')
);

function createSlideWithPlaceholders(pptx, layoutName, content) {
  const layout = templates.layouts.find(l => l.masterName === layoutName);
  if (!layout) throw new Error(`Layout "${layoutName}" not found`);
  
  const slide = pptx.addSlide({ masterName: layoutName });
  
  // Parse coordinates from code snippets
  const coords = {};
  layout.code.forEach(line => {
    const match = line.match(/ctx\.(\w+).*\{([^}]+)\}/);
    if (match) {
      const [, key, options] = match;
      const x = parseFloat(options.match(/x:\s*([\d.]+)/)?.[1]);
      const y = parseFloat(options.match(/y:\s*([\d.]+)/)?.[1]);
      const w = parseFloat(options.match(/w:\s*([\d.]+)/)?.[1]);
      const h = parseFloat(options.match(/h:\s*([\d.]+)/)?.[1]);
      coords[key] = { x, y, w, h };
    }
  });
  
  // Add content
  if (content.title && coords.title) {
    slide.addText(content.title, {
      ...coords.title,
      fontSize: content.titleSize || 36,
      bold: true
    });
  }
  
  if (content.subHeadline && coords.subHeadline) {
    slide.addText(content.subHeadline, {
      ...coords.subHeadline,
      fontSize: content.subSize || 18
    });
  }
  
  if (content.body && coords.body) {
    slide.addText(content.body, {
      ...coords.body,
      fontSize: content.bodySize || 16
    });
  }
  
  if (content.footnote && coords.footnote) {
    slide.addText(content.footnote, {
      ...coords.footnote,
      fontSize: 7,
      color: "666666"
    });
  }
  
  return slide;
}

// Usage
const pptx = new pptxgen();

createSlideWithPlaceholders(pptx, "Two Content", {
  title: "Benefits vs. Challenges",
  subHeadline: "Cloud Migration Analysis",
  body: [
    { text: "Scalability", options: { bullet: true } },
    { text: "Cost savings", options: { bullet: true } }
  ],
  footnote: "Source: IT Department"
});

await pptx.writeFile({ fileName: "output.pptx" });
```

## ‚ùì **FAQ**

### **Q: Why are there "placeholders" mentioned in the layout XML?**
A: The layout XML contains placeholder shapes that define the **visual template** (like logos, footers). When you create a slide with that layout, you get the visual design, but you **don't inherit the placeholders** - you add your own shapes.

### **Q: Why don't placeholders work like the demo in `demos/modules/demo_master.mjs`?**
A: That demo uses a **completely different system**. It defines slide masters programmatically with named placeholders. The 55 custom layouts in this repository are **pre-baked XML templates** without editable placeholders.

### **Q: What's the point of referencing a layout if I have to position everything manually?**
A: The layout provides:
- ‚úÖ **Consistent visual design** (backgrounds, borders, colors)
- ‚úÖ **Pre-positioned logos** (4 variants)
- ‚úÖ **Professional styling** matching comprehensive-test.pptx
- ‚úÖ **Theme colors and fonts**

You get the **look and feel**, but must manually add **content**.

### **Q: Can I use placeholder names from the layout XML?**
A: **No.** The `{ placeholder: "name" }` option in PptxGenJS only works with slide masters defined using `defineSlideMaster()` (which this repository doesn't use). For these custom layouts, you MUST use coordinates.

### **Q: Why does `comprehensive-test.pptx` look different when I edit it in PowerPoint?**
A: `comprehensive-test.pptx` is the **reference template** showing what the layouts look like. When you **open it in PowerPoint**, you can edit the layout placeholders. But when you **generate a PPTX with PptxGenJS**, the library creates slides with manually positioned content, not editable placeholders.

### **Q: Is there any way to use actual placeholders?**
A: Not with these 55 pre-defined layouts. You would need to:
1. Define slide masters programmatically using `pptx.defineSlideMaster()`
2. Add named placeholders to those masters
3. Then use `{ placeholder: "name" }` when adding content

This is a completely different approach and not compatible with the embedded layout system.

### **Q: How do I know which coordinates to use?**
A: Use `tools/step_templates.llm.json`. These coordinates are **suggestions** for where content aligns well with the layout's visual design (avoiding logos, margins, etc.).

### **Q: Can I modify the coordinates?**
A: **Yes!** The coordinates in `step_templates.llm.json` are **recommendations**, not requirements. You can position content anywhere. The suggested coordinates just ensure content:
- Avoids logo areas
- Respects margins
- Aligns with the layout's visual design

### **Q: What if I want to add extra content?**
A: You can add content **anywhere** using **any** coordinates. The layout just provides the background/logos. All your content is independent shapes.

### **Q: How do I list all available layouts?**
A:
```bash
cat tools/step_templates.llm.json | jq '.layouts[].masterName'
```

## üöÄ **Summary**

### **The Truth About "Placeholders" in This System**

1. **No editable placeholders** - These aren't traditional PowerPoint placeholders
2. **Layouts provide visual design** - You get backgrounds, logos, styling
3. **All content is manually positioned** - Use absolute `x, y, w, h` coordinates
4. **Coordinates are suggestions** - `step_templates.llm.json` provides recommended positions
5. **You control everything** - Full flexibility to position content anywhere
